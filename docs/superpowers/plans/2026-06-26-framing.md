# Framing / Message Protection (Plan 7 of 8) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement RFC 9420 §6 message framing and protection, gated by the official `message-protection.json` KAT. Concretely: the framing value types (`WireFormat`, `ContentType`, `Sender`, `FramedContent`, `FramedContentAuthData`, `AuthenticatedContent`, and the `MLSMessage` wrapper); the content-authentication signature over `FramedContentTBS` (§6.1); the `PublicMessage` membership tag `MAC(membership_key, AuthenticatedContentTBM)` (§6.2) plus its protect/unprotect (sign + MAC, verify signature + MAC); and the `PrivateMessage` AEAD protection (§6.3) — per-sender content encryption with the secret-tree ratchet key/nonce (§6.3.1, §9), the `reuse_guard` nonce masking, and the `SenderData` encryption under the `sender_data` key/nonce derived from `sender_data_secret` + a ciphertext sample (§6.3.2). Both message types must round-trip (`Protect`→serialize→parse→`Unprotect`) and must `Unprotect` the vendor-supplied `*_pub`/`*_priv` blobs back to the original `proposal`/`commit`/`application` content.

**Architecture:** A new package **`mls/framing`** holds all of this. Justification: framing is the layer that *composes* the lower primitives into authenticated/encrypted protocol messages. It consumes `mls/cipher` (`Suite.SignWithLabel`/`VerifyWithLabel`/`MAC`/`Seal`/`Open`), `mls/keyschedule` (`GroupContext`, `SecretTree.KeyNonce`, `SenderDataKeyNonce`, `RatchetType`), and `mls/tree` (`ProtocolVersionMLS10`, `ProposalType`, and the `DecodeLeafNode`/`DecodeUpdatePath` cursor decoders used to delimit inline proposal/commit content). It sits *above* all three and *below* the future group state machine. It cannot live in `keyschedule` (which must stay free of message-object concerns) nor in `tree`. **No import cycle:** `keyschedule` imports `cipher`+`tree`+`syntax`; `tree` imports `cipher`+`syntax`; `framing` imports `cipher`+`keyschedule`+`tree`+`syntax`; nothing imports `framing`. New types follow the established codec convention — unexported `marshal(*syntax.Builder) error` + package-level `decodeX(*syntax.Cursor) (X, error)` + exported `MarshalMLS()`/`UnmarshalMLS()` wrappers (the top-level `Unmarshal` enforces `Cursor.Empty()`).

This plan also makes three small, justified additions to existing packages (each folded into a task):
- **`mls/cipher`** (`aead.go`): `Suite.Seal(key, nonce, aad, plaintext)` / `Suite.Open(key, nonce, aad, ciphertext)` — raw AEAD seal/open. `PrivateMessage` needs AEAD with a *caller-supplied* key+nonce, which HPKE's `Seal`/`Open` do not expose. Implemented with `crypto/aes` + `crypto/cipher.NewGCM` for the AES-GCM AEADs (ids `0x0001`/`0x0002`); ChaCha20Poly1305 (id `0x0003`) is **not** in the stdlib and is rejected — both registered suites (`0x0001`, `0x0002`) use **AES-128-GCM**, so this suffices for every executed KAT case.
- **`mls/tree`**: exported `DecodeLeafNode` / `DecodeUpdatePath` cursor wrappers (one line each, mirroring the existing `tree.DecodeExtension`) so `framing` can delimit a `Commit`'s `optional<UpdatePath>` and an `Update` proposal's inline `LeafNode` when skimming content.
- **`mls/syntax`**: `Cursor.Rest()` returning the remaining unread bytes (aliasing) so the content delimiter can capture exactly the bytes a skim consumed.

**Tech Stack:** Go 1.26 standard library only — `crypto/aes`, `crypto/cipher` (`NewGCM`), `crypto/hmac` (`Equal` for constant-time tag compare), `crypto/ed25519`, `crypto/ecdsa`/`crypto/ecdh` (test-only signer construction), `encoding/binary`, `crypto/rand` (test-only `reuse_guard`), `errors`/`fmt`. Builds on `mls/syntax` (Builder/Cursor + `WriteOpaqueV`/`ReadOpaqueV`), `mls/cipher` (`Suite`, `Lookup`, `SignWithLabel`, `VerifyWithLabel`, `MAC`, `HashLen`, `AEADKeySize`, `AEADNonceSize`, new `Seal`/`Open`), `mls/keyschedule` (`GroupContext`, `SecretTree`/`NewSecretTree`/`KeyNonce`, `SenderDataKeyNonce`, `RatchetType`/`HandshakeRatchet`/`ApplicationRatchet`), `mls/tree` (`ProtocolVersion`, `ProtocolVersionMLS10`, `ProposalType`, `DecodeLeafNode`, `DecodeUpdatePath`), and `mls/internal/katutil` (`HexBytes`, `Load`).

**Spec reference:** RFC 9420 §6 (framing), §6.1 (content authentication / `FramedContentTBS`), §6.2 (`PublicMessage` / membership tag), §6.3 (`PrivateMessage`), §6.3.1 (content encryption), §6.3.2 (sender-data encryption), §9/§9.1 (secret tree ratchets — already implemented in Plan 5). KAT format: <https://github.com/mlswg/mls-implementations/blob/main/test-vectors.md> ("Message Protection").

## Design notes (read before implementing)

Every fact below was **verified byte-for-byte against the live `message-protection.json` (case 0, suite 1) during planning** by writing throwaway probes that (a) decoded `proposal_pub`/`commit_pub`/`application_priv`/`proposal_priv`, (b) re-derived the secret-tree key/nonce and AEAD-opened the `*_priv` ciphertexts back to the original content, and (c) recomputed the `FramedContentTBS` signature and `AuthenticatedContentTBM` membership tag. The probes were deleted and the working tree left clean. Get these exactly right or the KAT will not pass.

### N0. `MLSMessage` envelope (RFC 9420 §6) — every vector blob is a full `MLSMessage`
The `*_pub` and `*_priv` blobs are **not** bare `PublicMessage`/`PrivateMessage`; each is wrapped:
```
struct {
    ProtocolVersion version;          // uint16 = 0x0001 (mls10)
    WireFormat      wire_format;      // uint16
    select (wire_format) {
        case mls_public_message:  PublicMessage  public_message;
        case mls_private_message: PrivateMessage private_message;
        ...
    };
} MLSMessage;
```
`WireFormat`: `mls_public_message = 0x0001`, `mls_private_message = 0x0002` (also `welcome=0x0003`, `group_info=0x0004`, `key_package=0x0005`, out of scope). Verified prefixes: `proposal_pub`/`commit_pub` begin `0001 0001`; `application_priv`/`proposal_priv`/`commit_priv` begin `0001 0002`.

### N1. `Sender` (RFC 9420 §6) — variable body keyed on `sender_type`
```
struct {
    SenderType sender_type;           // uint8
    select (sender_type) {
        case member:             uint32 leaf_index;
        case external:           uint32 sender_index;
        case new_member_commit:                       // empty
        case new_member_proposal:                     // empty
    };
} Sender;
```
`SenderType`: `reserved=0`, `member=1`, `external=2`, `new_member_proposal=3`, `new_member_commit=4`. In every executed case the sender is `member` with `leaf_index = 1` (verified: the decrypted `SenderData` carries `leaf_index = 1`, and the `PublicMessage` framing encodes `01 00000001`).

### N2. `FramedContent` (RFC 9420 §6) — inline content, **no length prefix** for proposal/commit
```
struct {
    opaque       group_id<V>;
    uint64       epoch;
    Sender       sender;
    opaque       authenticated_data<V>;
    ContentType  content_type;        // uint8: application=1, proposal=2, commit=3
    select (content_type) {
        case application: opaque application_data<V>;   // LENGTH-PREFIXED
        case proposal:    Proposal proposal;            // inline, NO prefix
        case commit:      Commit   commit;              // inline, NO prefix
    };
} FramedContent;
```
**This is the central byte-accuracy fact.** Verified by decoding `proposal_pub`: after `content_type=02` the next 6 bytes are exactly the vector's `proposal` (`000300000002`) with no length prefix, immediately followed by `auth.signature`. For `application` the content **is** length-prefixed (`opaque<V>`). Therefore `FramedContent.Content` is stored as the *raw inner serialization* (the unwrapped `application_data` bytes, or the verbatim `Proposal`/`Commit` bytes), and the encoder writes `WriteOpaqueV` for application vs `WriteRaw` for proposal/commit.

**Content delimitation on decode** (no trailing-field length prefix to lean on): the proposal/commit content boundary is found by *skimming* the inner structure:
- `application`: `ReadOpaqueV` (self-delimiting).
- `commit`: `Commit = { ProposalOrRef proposals<V>; optional<UpdatePath> path; }`. `proposals<V>` is a varint-length-prefixed vector → `ReadOpaqueV` skips it without parsing its elements; then read the `optional` presence byte (and, if present, `tree.DecodeUpdatePath`). Verified on `commit_pub`: `proposals` body = 70 bytes, `path` presence = `00` (absent).
- `proposal`: `Proposal = { ProposalType msg_type<uint16>; select(body) }`. Skim the 2-byte type then the body: `remove(3)→uint32`; `external_init(6)→opaque<V>`; `group_context_extensions(7)→Extension<V>` (`opaque<V>` skip); `reinit(5)→ group_id<V> + uint16 + uint16 + Extension<V>`; `psk(4)→ PreSharedKeyID` (see N2a); `update(2)→ tree.DecodeLeafNode`; `add(1)→` embeds a `KeyPackage`, which is not implemented until Plan 8 (Welcome), so **return an explicit error** for Add-proposal framing. The KAT only exercises `remove`, so this is sufficient and honest.

The delimiter captures the consumed bytes via `Cursor.Rest()` (added in this plan): snapshot the remaining slice, run the skim, slice `rest[:before-Remaining()]`.

### N2a. `PreSharedKeyID` skim (RFC 9420 §8.4) — needed only by the proposal skimmer
```
struct {
    PSKType psktype;                  // uint8: external=1, resumption=2
    select (psktype) {
        case external:   opaque psk_id<V>;
        case resumption: ResumptionPSKUsage usage; opaque psk_group_id<V>; uint64 psk_epoch;
    };
    opaque psk_nonce<V>;
}
```

### N3. `FramedContentAuthData` (RFC 9420 §6.1) — confirmation_tag only for commits
```
struct {
    opaque signature<V>;
    select (content_type) {
        case commit: MAC confirmation_tag;            // == opaque<V>
        case application: case proposal: ;            // empty
    };
} FramedContentAuthData;
```
Verified: `proposal_pub` auth = `signature<V>` (64 bytes for Ed25519) only; `commit_pub` auth = `signature<V>` (64) + `confirmation_tag<V>` (32). The decoder must know `content_type` to decide whether to read `confirmation_tag` — pass it in.

### N4. `FramedContentTBS` and the signature (RFC 9420 §6.1) — label `"FramedContentTBS"`
```
struct {
    ProtocolVersion version = mls10;  // uint16 = 0x0001
    WireFormat      wire_format;      // uint16 (the enclosing message's wire format)
    FramedContent   content;
    select (content.sender.sender_type) {
        case member: case new_member_commit:  GroupContext context;   // serialized GroupContext
        case external: case new_member_proposal: ;                     // empty
    };
} FramedContentTBS;

signature = SignWithLabel(signature_key, "FramedContentTBS", FramedContentTBS)
```
Verified: `VerifyWithLabel(signature_pub, "FramedContentTBS", version||wire_format||framedContent||groupContext, signature)` returns **true** for `proposal_pub` and `commit_pub`, where `groupContext` is the serialized `keyschedule.GroupContext{ Version:mls10, CipherSuite, GroupID, Epoch, TreeHash, ConfirmedTranscriptHash, Extensions:nil }` built from the vector's `group_id`/`epoch`/`tree_hash`/`confirmed_transcript_hash` (empty extensions). The `GroupContext` block is present because the sender is `member`. `cipher.SignWithLabel`/`VerifyWithLabel` already prepend `"MLS 1.0 "` and frame `SignContent = { opaque label<V>; opaque content<V> }`.

### N5. `PublicMessage` + membership tag (RFC 9420 §6.2) — label `"membership_tag"` via raw MAC over TBM
```
struct {
    FramedContent          content;
    FramedContentAuthData  auth;
    select (content.sender.sender_type) {
        case member:  MAC membership_tag;             // == opaque<V>, ONLY for member
    };
} PublicMessage;

struct {                                              // AuthenticatedContentTBM
    FramedContentTBS       content_tbs;
    FramedContentAuthData  auth;
} AuthenticatedContentTBM;

membership_tag = MAC(membership_key, AuthenticatedContentTBM)
```
The `membership_tag` is **not** wrapped in `optional<>` — it is a bare `MAC` (`opaque<V>`) present iff `sender_type == member`. Verified by decoding `proposal_pub`: after `signature<V>` comes `20` (length 32) + 32-byte MAC, with nothing trailing. Verified: `MAC(membership_key, framedContentTBS || framedContentAuthData)` equals the vector's `membership_tag` byte-for-byte for both `proposal_pub` and `commit_pub`. Note the TBM is `FramedContentTBS` (which itself begins with version+wire_format and ends with the `GroupContext`) concatenated with the serialized `FramedContentAuthData` — **reuse the exact TBS bytes**, do not rebuild from `FramedContent` alone.

### N6. `PrivateMessage` (RFC 9420 §6.3) — field order
```
struct {
    opaque       group_id<V>;
    uint64       epoch;
    ContentType  content_type;        // uint8
    opaque       authenticated_data<V>;
    opaque       encrypted_sender_data<V>;
    opaque       ciphertext<V>;
} PrivateMessage;
```
Note the order differs from `FramedContent`: `content_type` comes **before** `authenticated_data` here. Verified by decoding `application_priv`: `group_id<V>`, `epoch=0x121212`, `content_type=01`, `authenticated_data<V>` (len 0), `encrypted_sender_data<V>` (len 28 = 12-byte `SenderData` + 16-byte GCM tag), `ciphertext<V>` (len 125), nothing trailing.

### N7. Content encryption (RFC 9420 §6.3.1) — ratchet selection + reuse_guard + AAD
```
PrivateContentAAD = struct {                          // AAD for the content AEAD
    opaque       group_id<V>;
    uint64       epoch;
    ContentType  content_type;
    opaque       authenticated_data<V>;
};

PrivateMessageContent = struct {                      // the AEAD plaintext
    select (content_type) {                           //   same inline rule as N2
        case application: opaque application_data<V>;
        case proposal:    Proposal proposal;
        case commit:      Commit   commit;
    };
    FramedContentAuthData auth;                        // signature [+ confirmation_tag]
    opaque padding[length_of_padding];                // all-zero, length not transmitted
};

(key, nonce) = SecretTree.KeyNonce(leaf_index, ratchet, generation)   // §9.1
reuse_guard  = 4 random bytes (in SenderData)
nonce'       = (nonce[0..3] XOR reuse_guard) || nonce[4..Nn-1]
ciphertext   = AEAD.Seal(key, nonce', PrivateContentAAD, PrivateMessageContent)
```
- **Ratchet selection:** `application` → `keyschedule.ApplicationRatchet`; `proposal`/`commit` → `keyschedule.HandshakeRatchet`. Verified: `application_priv` opens with the application ratchet, `proposal_priv` with the handshake ratchet, both at `leaf_index=1, generation=0` of `NewSecretTree(suite, 2, encryption_secret)`.
- **`reuse_guard`:** XOR the 4 guard bytes into the **first 4 bytes** of the ratchet nonce only. Verified: AEAD `Open` succeeds only with this masking.
- **Padding:** the trailing bytes of the decrypted `PrivateMessageContent` after `content` + `auth` must be **all zero**; their count is not transmitted (consume to end of plaintext). Verified: padding length 0 in case 0. `Protect` takes a caller-chosen `paddingSize` (the KAT round-trip uses 0).
- **Plaintext layout** matches N2's inline rule (application length-prefixed; proposal/commit raw), then `FramedContentAuthData` (N3), then padding. Verified: `application_priv` plaintext = `opaque<V>(application)` + `signature<V>` + (0 padding); `proposal_priv` plaintext = `000300000002` + `signature<V>` + (0 padding).

### N8. Sender-data encryption (RFC 9420 §6.3.2) — sample, key/nonce, AAD
```
SenderData = struct {
    uint32 leaf_index;
    uint32 generation;
    opaque reuse_guard[4];            // fixed 4 bytes, NO length prefix
};

SenderDataAAD = struct {
    opaque       group_id<V>;
    uint64       epoch;
    ContentType  content_type;
};

ciphertext_sample     = ciphertext[0 .. KDF.Nh-1]    (whole ciphertext if shorter)
sender_data_key       = ExpandWithLabel(sender_data_secret, "key",   ciphertext_sample, AEAD.Nk)
sender_data_nonce     = ExpandWithLabel(sender_data_secret, "nonce", ciphertext_sample, AEAD.Nn)
encrypted_sender_data = AEAD.Seal(sender_data_key, sender_data_nonce, SenderDataAAD, SenderData)
```
The key/nonce derivation is **already implemented** as `keyschedule.SenderDataKeyNonce(suite, sender_data_secret, ciphertext)` (Plan 5), which takes the full ciphertext and samples internally. Verified: opening `application_priv.encrypted_sender_data` yields `SenderData{leaf_index:1, generation:0, reuse_guard:d8c6e43a}` and the recovered guard/generation then open the content. `SenderData` is a fixed 12-byte struct (`reuse_guard` is `opaque[4]`, an array — **no** length prefix).

### N9. KAT group parameters — pin `nLeaves = 2`
The Message-Protection KAT is defined over a **2-member group with the sender at leaf index 1**. The vector does not carry a tree size; build the secret tree as `keyschedule.NewSecretTree(suite, 2, encryption_secret)`. Verified: only `nLeaves = 2` reproduces the leaf-1 ratchet key/nonce that opens the `*_priv` ciphertexts.

### N10. KAT verification approach (randomized encryption ⇒ unprotect + round-trip)
`PublicMessage` and `PrivateMessage` protection are randomized (fresh signatures vary for ECDSA; `reuse_guard` and the AEAD are sender-chosen), so the KAT is **not** a byte-equality check on `Protect` output. Two checks per content kind:
1. **Unprotect the vendor blob:** parse `proposal_pub`/`commit_pub` → `UnprotectPublic` → recovered `FramedContent.Content` equals the vector's `proposal`/`commit`, and signature+membership_tag verify. Parse `application_priv`/`proposal_priv`/`commit_priv` → `UnprotectPrivate` → recovered content equals the vector's `application`/`proposal`/`commit`, and the signature verifies.
2. **Round-trip:** `Protect{Public,Private}` the vector's content with a signer built from `signature_priv` → `MarshalMLS` → `UnmarshalMLS` → `Unprotect{Public,Private}` → recovered content equals the input. (`PublicMessage` round-trip reuses the `confirmation_tag` recovered in step 1 for commits; `PrivateMessage` round-trip uses `generation=0`, random `reuse_guard`, `paddingSize=0`.)

Skip cases whose `cipher_suite` is unregistered (suites 3–7), and **fail if zero cases executed** (guard against an all-skip false pass — suites 1 and 2 are present and registered).

### N11. Test-only signer construction from `signature_priv`
Suite 1 (`Ed25519`): `signature_priv` is a 32-byte seed → `ed25519.NewKeyFromSeed(seed)` (a `crypto.Signer`). Suite 2 (`ECDSA-P256`): `signature_priv` is a big-endian P-256 scalar → build `*ecdsa.PrivateKey` from the scalar (`ecdh.P256().NewPrivateKey` to validate the scalar, then set `D`/`PublicKey` for `ecdsa`). The same pattern is used by the TreeKEM KAT's `buildSigner` (Plan 6, N8); reuse that approach in the framing KAT test.

- [ ] **`mls/cipher/aead.go`** (implement):
  ```go
  package cipher

  import (
  	"crypto/aes"
  	"crypto/cipher"
  	"fmt"
  )

  // aeadCipher builds the suite's AEAD with the given key. Only the AES-GCM
  // AEADs (RFC 9180 ids 0x0001 AES-128-GCM and 0x0002 AES-256-GCM) are
  // supported by the standard library; ChaCha20Poly1305 (0x0003) is not in the
  // stdlib and is rejected. crypto/cipher.NewGCM uses the standard 12-byte
  // nonce, matching Suite.AEADNonceSize.
  func (s Suite) aeadCipher(key []byte) (cipher.AEAD, error) {
  	switch s.aead.ID() {
  	case 0x0001, 0x0002:
  		block, err := aes.NewCipher(key)
  		if err != nil {
  			return nil, err
  		}
  		return cipher.NewGCM(block)
  	default:
  		return nil, fmt.Errorf("cipher: raw AEAD seal/open unsupported for AEAD id %#x", s.aead.ID())
  	}
  }

  // Seal encrypts plaintext under the suite's AEAD with the caller-supplied key,
  // nonce, and associated data (RFC 9420 §6.3 PrivateMessage). The returned
  // ciphertext includes the AEAD tag.
  func (s Suite) Seal(key, nonce, aad, plaintext []byte) ([]byte, error) {
  	a, err := s.aeadCipher(key)
  	if err != nil {
  		return nil, err
  	}
  	return a.Seal(nil, nonce, plaintext, aad), nil
  }

  // Open decrypts and authenticates ciphertext under the suite's AEAD with the
  // caller-supplied key, nonce, and associated data. It returns an error if
  // authentication fails.
  func (s Suite) Open(key, nonce, aad, ciphertext []byte) ([]byte, error) {
  	a, err := s.aeadCipher(key)
  	if err != nil {
  		return nil, err
  ```

  ```go
  // (opaque<V>); Proposal/Commit are written verbatim/inline (RFC 9420 §6).
  func marshalContent(b *syntax.Builder, ct ContentType, content []byte) error {
  	switch ct {
  	case ContentTypeApplication:
  		return b.WriteOpaqueV(content)
  	case ContentTypeProposal, ContentTypeCommit:

  // decodeContent reads the inner content, returning the raw inner serialization.
  // For proposal/commit (no length prefix) it skims the structure to find the
  // boundary, capturing exactly the consumed bytes via Cursor.Rest.
  func decodeContent(c *syntax.Cursor, ct ContentType) ([]byte, error) {
  	switch ct {
  	case ContentTypeApplication:
  		return c.ReadOpaqueV()
  	case ContentTypeProposal:
  		return skim(c, skimProposal)
  	case ContentTypeCommit:
  		return skim(c, skimCommit)
  	default:
  		return nil, fmt.Errorf("framing: invalid content type %d", ct)
  	}
  }

  // skim runs f to advance c over an inline structure and returns the consumed
  // bytes.
  func skim(c *syntax.Cursor, f func(*syntax.Cursor) error) ([]byte, error) {
  	rest := c.Rest()
  	before := len(rest)
  	if err := f(c); err != nil {
  		return nil, err
  	}
  	return rest[:before-c.Remaining()], nil
  }

  // skimCommit advances over Commit = { ProposalOrRef proposals<V>;
  // optional<UpdatePath> path; } (RFC 9420 §12.4).
  func skimCommit(c *syntax.Cursor) error {
  	if _, err := c.ReadOpaqueV(); err != nil { // proposals<V>
  		return err
  	}
  	present, err := c.ReadUint8()

  		return err
  	}
  	switch present {
  	case 0:
  		return nil
  	case 1:
  		_, err := tree.DecodeUpdatePath(c)
  		return err
  	default:
  		return fmt.Errorf("framing: invalid optional<UpdatePath> presence %d", present)
  	}
  }

  // (RFC 9420 §6/§12.1). Add proposals embed a KeyPackage, which framing does
  // not yet parse (Plan 8 / Welcome); they are rejected.
  func skimProposal(c *syntax.Cursor) error {
  	pt, err := c.ReadUint16()

  			return err

  			return err
  		}

  			return err
  		}
  ```

  ```go
  	ac := AuthenticatedContent{WireFormat: WireFormatPublicMessage, Content: fc}
  	if fc.ContentType == ContentTypeCommit {
  		ac.Auth.ConfirmationTag = confirmationTag
  	}
  	if err := ac.sign(suite, signer, gc); err != nil {
  		return PublicMessage{}, err
  	}
  	m := PublicMessage{Content: ac.Content, Auth: ac.Auth}
  	if fc.Sender.Type == SenderTypeMember {
  		tbm, err := authenticatedContentTBM(WireFormatPublicMessage, fc, ac.Auth, gc)
  		if err != nil {
  			return PublicMessage{}, err
  		}
  		m.MembershipTag = suite.MAC(membershipKey, tbm)
  	}
  	return m, nil
  }

  // UnprotectPublic verifies a PublicMessage's membership tag (member senders)
  // and signature, returning the authenticated content (RFC 9420 §6.2).
  func UnprotectPublic(suite cipher.Suite, signaturePub []byte, gc *keyschedule.GroupContext, membershipKey []byte, m PublicMessage) (AuthenticatedContent, error) {
  	if m.Content.Sender.Type == SenderTypeMember {
  		tbm, err := authenticatedContentTBM(WireFormatPublicMessage, m.Content, m.Auth, gc)
  		if err != nil {
  			return AuthenticatedContent{}, err
  		}
  		if !hmac.Equal(suite.MAC(membershipKey, tbm), m.MembershipTag) {
  			return AuthenticatedContent{}, errors.New("framing: membership_tag verification failed")
  		}
  	}
  	ac := AuthenticatedContent{WireFormat: WireFormatPublicMessage, Content: m.Content, Auth: m.Auth}
  	if !ac.verify(suite, signaturePub, gc) {
  		return AuthenticatedContent{}, errors.New("framing: signature verification failed")
  	}
  	return ac, nil
  }
  ```

  ```go
  package framing

  import (
  	"crypto"
  	"errors"
  	"fmt"

  	"github.com/trevex/mls-mlkem-go/mls/cipher"
  	"github.com/trevex/mls-mlkem-go/mls/keyschedule"
  	"github.com/trevex/mls-mlkem-go/mls/syntax"
  )

  // PrivateMessage is an AEAD-protected framed message (RFC 9420 §6.3).
  type PrivateMessage struct {
  	GroupID             []byte
  	Epoch               uint64
  	ContentType         ContentType
  	AuthenticatedData   []byte
  	EncryptedSenderData []byte
  	Ciphertext          []byte
  }

  func (m PrivateMessage) marshal(b *syntax.Builder) error {
  	if err := b.WriteOpaqueV(m.GroupID); err != nil {
  		return err
  	}
  	b.WriteUint64(m.Epoch)
  	b.WriteUint8(uint8(m.ContentType))
  	if err := b.WriteOpaqueV(m.AuthenticatedData); err != nil {
  		return err
  	}
  	if err := b.WriteOpaqueV(m.EncryptedSenderData); err != nil {
  		return err
  	}
  	return b.WriteOpaqueV(m.Ciphertext)
  }

  func decodePrivateMessage(c *syntax.Cursor) (PrivateMessage, error) {
  	var m PrivateMessage
  	var err error
  	if m.GroupID, err = c.ReadOpaqueV(); err != nil {
  		return m, err
  	}
  	if m.Epoch, err = c.ReadUint64(); err != nil {
  		return m, err
  	}
  	ct, err := c.ReadUint8()
  	if err != nil {
  		return m, err
  	}
  	m.ContentType = ContentType(ct)
  	if m.AuthenticatedData, err = c.ReadOpaqueV(); err != nil {
  		return m, err
  	}
  	if m.EncryptedSenderData, err = c.ReadOpaqueV(); err != nil {
  		return m, err
  	}
  	if m.Ciphertext, err = c.ReadOpaqueV(); err != nil {
  		return m, err
  	}
  	return m, nil
  }

  // senderData is the per-message sender metadata encrypted into a PrivateMessage
  // (RFC 9420 §6.3.2). reuse_guard is a fixed 4-byte array (no length prefix).
  type senderData struct {
  	LeafIndex  uint32
  	Generation uint32
  	ReuseGuard [4]byte
  }

  func (sd senderData) marshal(b *syntax.Builder) {
  	b.WriteUint32(sd.LeafIndex)
  	b.WriteUint32(sd.Generation)
  	b.WriteRaw(sd.ReuseGuard[:])
  }

  func decodeSenderData(c *syntax.Cursor) (senderData, error) {
  	var sd senderData
  	var err error
  	if sd.LeafIndex, err = c.ReadUint32(); err != nil {
  		return sd, err
  	}
  	if sd.Generation, err = c.ReadUint32(); err != nil {
  		return sd, err
  	}
  	g, err := c.ReadRaw(4)
  	if err != nil {
  		return sd, err
  	}
  	copy(sd.ReuseGuard[:], g)
  	return sd, nil
  }

  func senderDataAAD(groupID []byte, epoch uint64, ct ContentType) ([]byte, error) {
  	b := syntax.NewBuilder()
  	if err := b.WriteOpaqueV(groupID); err != nil {
  		return nil, err
  	}
  	b.WriteUint64(epoch)
  	b.WriteUint8(uint8(ct))
  	return b.Bytes(), nil
  }

  func privateContentAAD(groupID []byte, epoch uint64, ct ContentType, ad []byte) ([]byte, error) {
  	b := syntax.NewBuilder()
  	if err := b.WriteOpaqueV(groupID); err != nil {
  		return nil, err
  	}
  	b.WriteUint64(epoch)
  	b.WriteUint8(uint8(ct))
  	if err := b.WriteOpaqueV(ad); err != nil {
  		return nil, err
  	}
  	return b.Bytes(), nil
  }

  // ratchetTypeFor selects the secret-tree ratchet for a content type
  // (RFC 9420 §6.3.1/§9.1): application -> application ratchet; proposal/commit
  // (handshake) -> handshake ratchet.
  func ratchetTypeFor(ct ContentType) keyschedule.RatchetType {
  	if ct == ContentTypeApplication {
  		return keyschedule.ApplicationRatchet
  	}
  	return keyschedule.HandshakeRatchet
  }

  // applyReuseGuard XORs the 4-byte reuse_guard into the first 4 bytes of the
  // ratchet nonce (RFC 9420 §6.3.1).
  func applyReuseGuard(nonce []byte, guard [4]byte) []byte {
  	n := append([]byte(nil), nonce...)
  	for i := 0; i < 4; i++ {
  		n[i] ^= guard[i]
  	}
  	return n
  }

  // privateMessageContent serializes content || auth || padding (RFC 9420 §6.3.1).
  func privateMessageContent(fc FramedContent, auth FramedContentAuthData, paddingSize int) ([]byte, error) {
  	b := syntax.NewBuilder()
  	if err := marshalContent(b, fc.ContentType, fc.Content); err != nil {
  		return nil, err
  	}
  	if err := auth.marshal(b, fc.ContentType); err != nil {
  		return nil, err
  	}
  	if paddingSize > 0 {
  		b.WriteRaw(make([]byte, paddingSize))
  	}
  	return b.Bytes(), nil
  }

  // decodePrivateMessageContent parses content || auth || zero-padding.
  func decodePrivateMessageContent(pt []byte, ct ContentType) ([]byte, FramedContentAuthData, error) {
  	c := syntax.NewCursor(pt)
  	content, err := decodeContent(c, ct)
  	if err != nil {
  		return nil, FramedContentAuthData{}, err
  	}
  	auth, err := decodeFramedContentAuthData(c, ct)
  	if err != nil {
  		return nil, FramedContentAuthData{}, err
  	}
  	for _, x := range c.Rest() {
  		if x != 0 {
  			return nil, FramedContentAuthData{}, errors.New("framing: non-zero padding in PrivateMessageContent")
  		}
  	}
  	return content, auth, nil
  }

  // ProtectPrivate encrypts fc as a PrivateMessage (RFC 9420 §6.3). The sender's
  // leaf index is taken from fc.Sender (must be a member). st provides the
  // content key/nonce at (leaf, ratchet, generation); senderDataSecret derives
  // the sender-data key/nonce. confirmationTag is used only for commit content.
  func ProtectPrivate(suite cipher.Suite, signer crypto.Signer, gc *keyschedule.GroupContext, st *keyschedule.SecretTree, senderDataSecret []byte, fc FramedContent, generation uint32, reuseGuard [4]byte, paddingSize int, confirmationTag []byte) (PrivateMessage, error) {
  	if fc.Sender.Type != SenderTypeMember {
  		return PrivateMessage{}, fmt.Errorf("framing: PrivateMessage requires a member sender, got %v", fc.Sender.Type)
  	}
  	// 1. Sign the content.
  	ac := AuthenticatedContent{WireFormat: WireFormatPrivateMessage, Content: fc}
  	if fc.ContentType == ContentTypeCommit {
  		ac.Auth.ConfirmationTag = confirmationTag
  	}
  	if err := ac.sign(suite, signer, gc); err != nil {
  		return PrivateMessage{}, err
  	}
  	// 2. Content AEAD with the ratchet key/nonce (guarded by reuse_guard).
  	key, nonce, err := st.KeyNonce(fc.Sender.LeafIndex, ratchetTypeFor(fc.ContentType), generation)
  	if err != nil {
  		return PrivateMessage{}, err
  	}
  	contentAAD, err := privateContentAAD(fc.GroupID, fc.Epoch, fc.ContentType, fc.AuthenticatedData)
  	if err != nil {
  		return PrivateMessage{}, err
  	}
  	pt, err := privateMessageContent(fc, ac.Auth, paddingSize)
  	if err != nil {
  		return PrivateMessage{}, err
  	}
  	ciphertext, err := suite.Seal(key, applyReuseGuard(nonce, reuseGuard), contentAAD, pt)
  	if err != nil {
  		return PrivateMessage{}, err
  	}
  	// 3. Sender-data AEAD (key/nonce sampled from the content ciphertext).
  	sdKey, sdNonce, err := keyschedule.SenderDataKeyNonce(suite, senderDataSecret, ciphertext)
  	if err != nil {
  		return PrivateMessage{}, err
  	}
  	sdAAD, err := senderDataAAD(fc.GroupID, fc.Epoch, fc.ContentType)
  	if err != nil {
  		return PrivateMessage{}, err
  	}
  	sdb := syntax.NewBuilder()
  	senderData{LeafIndex: fc.Sender.LeafIndex, Generation: generation, ReuseGuard: reuseGuard}.marshal(sdb)
  	encSD, err := suite.Seal(sdKey, sdNonce, sdAAD, sdb.Bytes())
  	if err != nil {
  		return PrivateMessage{}, err
  	}
  	return PrivateMessage{
  		GroupID:             fc.GroupID,
  		Epoch:               fc.Epoch,
  		ContentType:         fc.ContentType,
  		AuthenticatedData:   fc.AuthenticatedData,
  		EncryptedSenderData: encSD,
  		Ciphertext:          ciphertext,
  	}, nil
  }

  // UnprotectPrivate decrypts the sender data, then the content, reconstructs the
  // FramedContent, and verifies its signature (RFC 9420 §6.3). signaturePub maps
  // a sender leaf index to its signature public key (the verifier looks this up
  // from the ratchet tree in a live group).
  func UnprotectPrivate(suite cipher.Suite, signaturePub func(leafIndex uint32) ([]byte, error), gc *keyschedule.GroupContext, st *keyschedule.SecretTree, senderDataSecret []byte, m PrivateMessage) (AuthenticatedContent, error) {
  	// 1. Sender data.
  	sdKey, sdNonce, err := keyschedule.SenderDataKeyNonce(suite, senderDataSecret, m.Ciphertext)
  	if err != nil {
  		return AuthenticatedContent{}, err
  	}
  	sdAAD, err := senderDataAAD(m.GroupID, m.Epoch, m.ContentType)
  	if err != nil {
  		return AuthenticatedContent{}, err
  	}
  	sdPlain, err := suite.Open(sdKey, sdNonce, sdAAD, m.EncryptedSenderData)
  	if err != nil {
  		return AuthenticatedContent{}, fmt.Errorf("framing: sender data open: %w", err)
  	}
  	sdc := syntax.NewCursor(sdPlain)
  	sd, err := decodeSenderData(sdc)
  	if err != nil {
  		return AuthenticatedContent{}, err
  	}
  	if !sdc.Empty() {
  		return AuthenticatedContent{}, errors.New("framing: trailing bytes after SenderData")
  	}
  	// 2. Content.
  	key, nonce, err := st.KeyNonce(sd.LeafIndex, ratchetTypeFor(m.ContentType), sd.Generation)
  	if err != nil {
  		return AuthenticatedContent{}, err
  	}
  	contentAAD, err := privateContentAAD(m.GroupID, m.Epoch, m.ContentType, m.AuthenticatedData)
  	if err != nil {
  		return AuthenticatedContent{}, err
  	}
  	pt, err := suite.Open(key, applyReuseGuard(nonce, sd.ReuseGuard), contentAAD, m.Ciphertext)
  	if err != nil {
  		return AuthenticatedContent{}, fmt.Errorf("framing: content open: %w", err)
  	}
  	content, auth, err := decodePrivateMessageContent(pt, m.ContentType)
  	if err != nil {
  		return AuthenticatedContent{}, err
  	}
  	// 3. Reconstruct + verify.
  	fc := FramedContent{
  		GroupID:           m.GroupID,
  		Epoch:             m.Epoch,
  		Sender:            Sender{Type: SenderTypeMember, LeafIndex: sd.LeafIndex},
  		AuthenticatedData: m.AuthenticatedData,
  		ContentType:       m.ContentType,
  		Content:           content,
  	}
  	ac := AuthenticatedContent{WireFormat: WireFormatPrivateMessage, Content: fc, Auth: auth}
  	pub, err := signaturePub(sd.LeafIndex)
  	if err != nil {
  		return AuthenticatedContent{}, err
  	}
  	if !ac.verify(suite, pub, gc) {
  		return AuthenticatedContent{}, errors.New("framing: signature verification failed")
  	}
  	return ac, nil
  }
  ```

- **Add-proposal content framing is deferred here.** `skimProposal` rejects `add(1)` because it embeds a `KeyPackage` (a Welcome/Join object). - **Stateful ratchets.** `SecretTree.KeyNonce` is stateless (re-derives from generation 0). A live client must track per-leaf generation counters (and discard consumed keys) for forward secrecy; the passive-client KAT will exercise specific generations, so Plan 8 should add a stateful ratchet wrapper around `SecretTree`.
- **Signature-key lookup.** `UnprotectPrivate` takes a `func(leafIndex) ([]byte, error)` resolver; Plan 8 wires this to the ratchet tree's leaf signature keys, and similarly resolves the per-sender `signature_pub` for `UnprotectPublic`.
- **Wire formats still missing:** `welcome`/`group_info`/`key_package` MLSMessage bodies are unimplemented; Plan 8 adds them for the passive-client/join flow.
