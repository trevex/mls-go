# Group — Protocol Objects & Wire Round-Trip (Plan 8a of 8) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the RFC 9420 §10/§12.1/§12.4/§12.4.3.1 protocol *objects* that the group engine (Plan 8b) composes — `KeyPackage` (+ `KeyPackageTBS` sign/verify + `KeyPackageRef`), the `Proposal` enum and all seven proposal bodies, `ProposalOrRef` + `Commit`, `GroupInfo` (+ `GroupInfoTBS`), and the `Welcome` family (`GroupSecrets`, `PathSecret`, `EncryptedGroupSecrets`, `Welcome`) — plus the `MLSMessage` envelope handling for the `welcome`/`group_info`/`key_package` wire formats. This plan is gated by two KATs: the official **`messages.json`** (a pure encode→decode→re-encode byte-equality round-trip over every message type, now that all of them exist) and **`welcome.json`** (decrypt a vendor-supplied `Welcome` with a joiner's `init_priv`, recover `GroupInfo`, and verify the `GroupInfoTBS` signature against the supplied `signer_pub`). It also lands the small lower-package helpers the Plan 8b engine needs for these objects (a `keyschedule.GroupContext` cursor decoder, the welcome-secret key/nonce + joiner-based epoch derivation, a `framing.AuthenticatedContent` serializer, and a `framing` KeyPackage skimmer that finally wires Add-proposal framing through).

**Architecture:** A new package **`mls/group`** holds every protocol object above and (in Plan 8b) the `Group` state machine. Justification: these objects sit *above* framing — a `Commit`/`Proposal` is the *content* a `FramedContent` carries, a `Welcome`/`GroupInfo`/`KeyPackage` is an `MLSMessage` body — and they compose **all** lower packages: `cipher` (`SignWithLabel`/`RefHash`/`EncryptWithLabel`/`Seal`), `keyschedule` (`GroupContext`, `PSKSecret`, epoch derivation), `tree` (`LeafNode`, `RatchetTree`, `UpdatePath`, `Extension`), `framing` (`MLSMessage`, `WireFormat`, `AuthenticatedContent`), and `syntax`. **No import cycle:** `group` imports `framing`+`keyschedule`+`tree`+`cipher`+`syntax`; nothing imports `group` (verified: `framing`'s Add skimmer parses a KeyPackage *structurally* via `tree.DecodeLeafNode`, so framing never needs the `group.KeyPackage` type). New types follow the established codec convention — unexported `marshal(*syntax.Builder) error` + package-level `decodeX(*syntax.Cursor) (X, error)` + exported `MarshalMLS()`/`UnmarshalMLS()` wrappers (the top-level `Unmarshal*` enforces `Cursor.Empty()`).

This plan also makes four small, justified additions to existing packages (each folded into Task 0):
- **`mls/keyschedule`** (`context.go`): exported `func DecodeGroupContext(c *syntax.Cursor) (GroupContext, error)` (one line, mirroring `tree.DecodeExtension`) — `GroupInfo`/`GroupInfoTBS` embed a `GroupContext` *followed by more fields*, so the cursor form is required (the existing `UnmarshalMLS` rejects trailing bytes).
- **`mls/keyschedule`** (`schedule.go`): `WelcomeKeyNonce(suite, welcomeSecret)` and `EpochSecretsFromJoiner(suite, joinerSecret, pskSecret, groupContext)` — the Welcome path derives epoch secrets from a *given* `joiner_secret` (not from `init_secret`+`commit_secret`), and decrypts `GroupInfo` with a key/nonce expanded from `welcome_secret`. Refactor `DeriveEpochSecrets` to call `EpochSecretsFromJoiner` (DRY).
- **`mls/framing`** (`framing.go`): exported `func (ac AuthenticatedContent) MarshalMLS() ([]byte, error)` serializing `wire_format || FramedContent || FramedContentAuthData` — Plan 8b feeds this into `keyschedule.SplitAuthenticatedContent` to derive the confirmed transcript hash.
- **`mls/framing`** (`framing.go`): replace the `skimProposal` Add stub (`return fmt.Errorf("... Plan 8 ...")`) with a real `skimKeyPackage` that walks `version, cipher_suite, init_key<V>, LeafNode (tree.DecodeLeafNode), extensions<V>, signature<V>` — wiring Add-proposal framing through without importing `group`.

**Tech Stack:** Go 1.26 standard library only — `crypto/ed25519`, `crypto/ecdsa`/`crypto/ecdh`/`crypto/elliptic` (test-only signer construction), `crypto/hmac` (`Equal`), `errors`/`fmt`, `bytes`. ## Design notes (read before implementing)

Every wire fact below was **verified byte-for-byte during planning** against the live `passive-client-welcome.json` and `welcome.json` vectors (suite 1 & 2) with throwaway probes that decoded the `welcome`/`key_package` blobs, recomputed the `KeyPackageRef`, and round-tripped the structures. The probes were deleted and the working tree left clean. Get these exactly right or the KATs will not pass.

### N0. `MLSMessage` envelopes for the three new wire formats (RFC 9420 §6/§12.4.3.1)
Every `welcome`, `key_package`, and `mls_group_info` blob is a full `MLSMessage`: `uint16 version (=0x0001 mls10) || uint16 wire_format || body`. Verified prefixes:
- `key_package` (passive-client + `messages.json` `mls_key_package`) begins `0001 0005` → body is a bare `KeyPackage`.
- `welcome` begins `0001 0003` → body is a bare `Welcome`.
- `mls_group_info` begins `0001 0004` → body is a bare `GroupInfo`.

`framing.WireFormat` already defines `WireFormatWelcome=0x0003`, `WireFormatGroupInfo=0x0004`, `WireFormatKeyPackage=0x0005`. `framing.MLSMessage` only carries `Public`/`Private`; rather than extend it (which would force a `framing→group` import cycle), the `group` package provides its own envelope readers/writers (`encodeMessage`/`decodeMessage`, §N7) for these three formats. `messages.json` also supplies **bare** (un-enveloped) objects: `ratchet_tree`, `group_secrets`, `add_proposal`, …, `commit` — these round-trip via the object's own `MarshalMLS`/`UnmarshalMLS`.

### N1. `KeyPackage` (RFC 9420 §10) — exact layout, TBS, and Ref
```
struct {
    ProtocolVersion version;          // uint16
    CipherSuite     cipher_suite;     // uint16
    HPKEPublicKey   init_key;         // opaque<V>
    LeafNode        leaf_node;        // inline, NO length prefix (tree.LeafNode wire form)
    Extension       extensions<V>;
    /* SignWithLabel(sig_key, "KeyPackageTBS", KeyPackageTBS) */
    opaque          signature<V>;
} KeyPackage;

struct {                             // KeyPackageTBS = KeyPackage minus signature
    ProtocolVersion version;
    CipherSuite     cipher_suite;
    HPKEPublicKey   init_key;
    LeafNode        leaf_node;
    Extension       extensions<V>;
} KeyPackageTBS;
```
- `leaf_node` is embedded **inline** (its own `tree.LeafNode` wire form, signature-included) with no extra length prefix — write `leafNode.MarshalMLS()` via `WriteRaw`, decode via `tree.DecodeLeafNode`.
- `signature = SignWithLabel(leaf_node.signature_key, "KeyPackageTBS", KeyPackageTBS)`; verify with `VerifyWithLabel`. `cipher.SignWithLabel` already prepends `"MLS 1.0 "` and frames `SignContent`.
- **`KeyPackageRef = RefHash("MLS 1.0 KeyPackage Reference", KeyPackage)`** where the value hashed is the **full `KeyPackage` struct serialization** (NOT the `MLSMessage` envelope). **Verified:** for `passive-client-welcome.json` case 0, `RefHash("MLS 1.0 KeyPackage Reference", keyPackageBytes)` equals the `Welcome`'s `EncryptedGroupSecrets.new_member` byte-for-byte, where `keyPackageBytes = mlsMessageBlob[4:]` (the envelope stripped). `cipher.RefHash` takes the **full** label verbatim (it does NOT add the `"MLS 1.0 "` prefix), so pass `"MLS 1.0 KeyPackage Reference"` literally.

### N2. `Proposal` + bodies (RFC 9420 §12.1)
```
struct {
    ProposalType proposal_type;       // uint16
    select (proposal_type) {
        case add:                      Add                     add;
        case update:                   Update                  update;
        case remove:                   Remove                  remove;
        case psk:                      PreSharedKey            preshared_key;
        case reinit:                   ReInit                  reinit;
        case external_init:            ExternalInit            external_init;
        case group_context_extensions: GroupContextExtensions  group_context_extensions;
    };
} Proposal;

struct { KeyPackage key_package; }                                      Add;            // type 1
struct { LeafNode  leaf_node;   }                                       Update;         // type 2
struct { uint32    removed;     }                                       Remove;         // type 3
struct { PreSharedKeyID psk;    }                                       PreSharedKey;   // type 4
struct { opaque group_id<V>; ProtocolVersion version;                                   // type 5
         CipherSuite cipher_suite; Extension extensions<V>; }           ReInit;
struct { opaque kem_output<V>; }                                       ExternalInit;   // type 6
struct { Extension extensions<V>; }                                    GroupContextExtensions; // type 7
```
`ProposalType` ids (verbatim): `add=1, update=2, remove=3, psk=4, reinit=5, external_init=6, group_context_extensions=7`. `Add` embeds a full inline `KeyPackage` (N1, no length prefix). `Update` embeds an inline `tree.LeafNode`. `PreSharedKey` embeds a `keyschedule.PreSharedKeyID` — **needs an exported `keyschedule.DecodePreSharedKeyID` cursor decoder** (the type's `decodePreSharedKeyID` is unexported); add it in Task 0 alongside `DecodeGroupContext`, and an exported `(PreSharedKeyID).MarshalTo(b)` for the encoder. `ProposalRef = RefHash("MLS 1.0 Proposal Reference", Proposal)`.

### N3. `ProposalOrRef` + `Commit` (RFC 9420 §12.4)
```
struct {
    ProposalOrRefType type;           // uint8: proposal=1, reference=2
    select (type) {
        case proposal:  Proposal    proposal;     // inline, no length prefix
        case reference: ProposalRef reference;    // opaque<V> (HashReference)
    };
} ProposalOrRef;

struct {
    ProposalOrRef proposals<V>;
    optional<UpdatePath> path;        // 0x00 absent / 0x01 + tree.UpdatePath
} Commit;
```
`ProposalRef` is a `HashReference = opaque<V>`. The `Commit` is exactly what `framing.skimCommit` already delimits (proposals as `opaque<V>` skip + optional path) — Plan 8b parses the *contents* via `group.decodeCommit`.

### N4. `GroupInfo` + `GroupInfoTBS` (RFC 9420 §12.4.3.1) — label `"GroupInfoTBS"`
```
struct {
    GroupContext group_context;       // keyschedule.GroupContext (inline)
    Extension    extensions<V>;        // carries ratchet_tree (0x0002) and external_pub (0x0004)
    MAC          confirmation_tag;     // opaque<V>
    uint32       signer;               // leaf index of the GroupInfo signer
    /* SignWithLabel(signer_leaf.signature_key, "GroupInfoTBS", GroupInfoTBS) */
    opaque       signature<V>;
} GroupInfo;

struct {                              // GroupInfoTBS = GroupInfo minus signature
    GroupContext group_context;
    Extension    extensions<V>;
    MAC          confirmation_tag;
    uint32       signer;
} GroupInfoTBS;
```
`signature = SignWithLabel(., "GroupInfoTBS", GroupInfoTBS)`. **Verified** (welcome.json): `VerifyWithLabel(signer_pub, "GroupInfoTBS", groupInfoTBS, signature)` returns **true** for the recovered `GroupInfo`. The `GroupContext` block is read with the new `keyschedule.DecodeGroupContext`. Extension type ids: `ratchet_tree = 0x0002` (**verified** — the passive-client `GroupInfo` carries the tree in a `0x0002` extension), `external_pub = 0x0004`.

### N5. `Welcome` family (RFC 9420 §12.4.3.1) — HPKE labels `"Welcome"`
```
struct { opaque path_secret<V>; } PathSecret;

struct {
    opaque         joiner_secret<V>;
    optional<PathSecret> path_secret;     // 0x00 absent / 0x01 + PathSecret
    PreSharedKeyID psks<V>;
} GroupSecrets;

struct {
    KeyPackageRef  new_member;            // opaque<V> (HashReference, == KeyPackageRef of recipient)
    HPKECiphertext encrypted_group_secrets;  // { opaque kem_output<V>; opaque ciphertext<V>; }
} EncryptedGroupSecrets;

struct {
    CipherSuite           cipher_suite;   // uint16
    EncryptedGroupSecrets secrets<V>;
    opaque                encrypted_group_info<V>;
} Welcome;
```
**Verified decryption path (this is the byte-accuracy crux for Plan 8b's join):**
- `GroupSecrets` is recovered with HPKE: `DecryptWithLabel(init_priv, "Welcome", context = encrypted_group_info, kem_output, ciphertext)`. The HPKE **context is the `Welcome.encrypted_group_info` field** (not empty, not the GroupInfo plaintext). The matching `EncryptedGroupSecrets` entry is the one whose `new_member == our KeyPackageRef`.
- `GroupInfo` is recovered with the **welcome key schedule**: `member_secret = Extract(joiner_secret, psk_secret)`; `welcome_secret = DeriveSecret(member_secret, "welcome")`; `welcome_key = ExpandWithLabel(welcome_secret, "key", "", AEAD.Nk)`; `welcome_nonce = ExpandWithLabel(welcome_secret, "nonce", "", AEAD.Nn)`; `group_info = AEAD.Open(welcome_key, welcome_nonce, aad="", encrypted_group_info)`. **The GroupInfo AEAD AAD is empty.** (All verified across 16 passive-client-welcome cases, suites 1 & 2 — see Plan 8b N-notes.)

`HPKECiphertext` reuses `tree.HPKECiphertext` (`{ kem_output<V>, ciphertext<V> }`); its `marshal`/`decode` are unexported, so add exported `tree.DecodeHPKECiphertext` + `(HPKECiphertext).MarshalTo` in Task 0 (mirroring `Extension`).

### N6. `welcome.json` KAT shape
`welcome.json` (7 cases, suites 1–7; execute 1 & 2) carries `cipher_suite`, `key_package` (MLSMessage), `init_priv`, `signer_pub`, `welcome` (MLSMessage). The test: parse the `welcome` envelope → `Welcome`; compute our `KeyPackageRef` from `key_package[4:]`; select the matching `EncryptedGroupSecrets`; `DecryptWithLabel(init_priv, "Welcome", encrypted_group_info, …)` → `GroupSecrets`; derive `welcome_key`/`welcome_nonce` (N5); `Open` → `GroupInfo`; `VerifyWithLabel(signer_pub, "GroupInfoTBS", groupInfoTBS, gi.Signature)` must be **true**. (welcome.json has no `epoch_authenticator`; the full join → authenticator is the passive-client KAT in Plan 8b.)

### N7. `messages.json` KAT shape — pure wire round-trip
`messages.json` (300 cases, all suites; **no per-case `cipher_suite` gate** — the blobs are suite-agnostic TLV, so execute every case) has one hex field per object: `mls_key_package`, `mls_welcome`, `mls_group_info` (enveloped MLSMessages), `key_package`?/`ratchet_tree`/`group_secrets`/`add_proposal`/`update_proposal`/`remove_proposal`/`pre_shared_key_proposal`/`re_init_proposal`/`external_init_proposal`/`group_context_extensions_proposal`/`commit` (bare objects), plus `public_message_*`/`private_message` (framing — already covered by the framing KAT, re-checked here for completeness). For each present field: `Unmarshal` the hex → `Marshal` → assert **byte-equal** to the input. This is the comprehensive structural gate proving every object's codec is exact. **Pin:** the `*_proposal` fields are **bare `Proposal`** objects (type tag + body), `commit` is a bare `Commit`, `group_secrets` is a bare `GroupSecrets`, `ratchet_tree` is the `optional<Node><V>` wire form (`tree.RatchetTree`, which needs a suite — use suite 1 for the round-trip since the bytes are suite-independent, or re-serialize via a raw passthrough; prefer parsing with suite 1 and re-marshaling).

  ```go
  // WelcomeKeyNonce derives the AEAD key/nonce that protect a Welcome's
  // encrypted_group_info (RFC 9420 §12.4.3.1):
  //   welcome_key   = ExpandWithLabel(welcome_secret, "key",   "", AEAD.Nk)
  //   welcome_nonce = ExpandWithLabel(welcome_secret, "nonce", "", AEAD.Nn)
  func WelcomeKeyNonce(suite cipher.Suite, welcomeSecret []byte) (key, nonce []byte, err error) {
  	key, err = suite.ExpandWithLabel(welcomeSecret, "key", nil, suite.AEADKeySize())
  	if err != nil {
  		return nil, nil, err
  	}
  	nonce, err = suite.ExpandWithLabel(welcomeSecret, "nonce", nil, suite.AEADNonceSize())
  	if err != nil {
  		return nil, nil, err
  	}
  	return key, nonce, nil
  }

  // EpochSecretsFromJoiner runs the §8 key schedule from a known joiner_secret
  // (the Welcome path: the joiner receives joiner_secret directly, rather than
  // deriving it from init_secret+commit_secret). A nil pskSecret is the all-zero
  // KDF.Nh vector.
  func EpochSecretsFromJoiner(suite cipher.Suite, joinerSecret, pskSecret, groupContext []byte) (EpochSecrets, error) {
  	nh := suite.HashLen()
  	if pskSecret == nil {
  		pskSecret = make([]byte, nh)
  	}
  	member, err := suite.Extract(joinerSecret, pskSecret) // Extract(salt=joiner_secret, IKM=psk_secret)
  	if err != nil {
  		return EpochSecrets{}, err
  	}
  	welcome, err := suite.DeriveSecret(member, "welcome")
  	if err != nil {
  		return EpochSecrets{}, err
  	}
  	epoch, err := suite.ExpandWithLabel(member, "epoch", groupContext, nh)
  	if err != nil {
  		return EpochSecrets{}, err
  	}
  	es := EpochSecrets{JoinerSecret: joinerSecret, WelcomeSecret: welcome, EpochSecret: epoch}
  	for _, d := range []struct {
  		label string
  		out   *[]byte
  	}{
  		{"sender data", &es.SenderDataSecret}, {"encryption", &es.EncryptionSecret},
  		{"exporter", &es.ExporterSecret}, {"external", &es.ExternalSecret},
  		{"confirm", &es.ConfirmationKey}, {"membership", &es.MembershipKey},
  		{"resumption", &es.ResumptionPSK}, {"authentication", &es.EpochAuthenticator},
  		{"init", &es.InitSecret},
  	} {
  		v, err := suite.DeriveSecret(epoch, d.label)
  		if err != nil {
  			return EpochSecrets{}, err
  		}
  		*d.out = v
  	}
  	return es, nil
  }
  ```

  ```go
  // skimKeyPackage advances over a KeyPackage (RFC 9420 §10), delimiting an
  // Add proposal's inline KeyPackage without importing the group package.
  func skimKeyPackage(c *syntax.Cursor) error {
  	if _, err := c.ReadUint16(); err != nil { // version
  		return err
  	}
  	if _, err := c.ReadUint16(); err != nil { // cipher_suite
  		return err
  	}
  	if _, err := c.ReadOpaqueV(); err != nil { // init_key<V>
  		return err
  	}
  	if _, err := tree.DecodeLeafNode(c); err != nil { // leaf_node (inline)
  		return err
  	}
  	if _, err := syntax.ReadVectorV(c, tree.DecodeExtension); err != nil { // extensions<V>
  		return err
  	}
  	_, err := c.ReadOpaqueV() // signature<V>
  	return err
  ```

  ```go
  // Package group implements the RFC 9420 §10/§12 protocol objects (KeyPackage,
  // Proposal, Commit, GroupInfo, Welcome) and (Plan 8b) the group state machine.
  package group

  import (
  	"fmt"

  	"github.com/trevex/mls-mlkem-go/mls/cipher"
  	"github.com/trevex/mls-mlkem-go/mls/syntax"
  	"github.com/trevex/mls-mlkem-go/mls/tree"
  )

  // KeyPackage advertises a client's init key + leaf node for being added to a
  // group (RFC 9420 §10).
  type KeyPackage struct {
  	Version     tree.ProtocolVersion
  	CipherSuite cipher.CipherSuite
  	InitKey     []byte // HPKEPublicKey opaque<V>
  	LeafNode    tree.LeafNode
  	Extensions  []tree.Extension
  	Signature   []byte // opaque<V>
  }

  // marshalTBS writes KeyPackageTBS (everything above the signature).
  func (kp KeyPackage) marshalTBS(b *syntax.Builder) error {
  	b.WriteUint16(uint16(kp.Version))
  	b.WriteUint16(uint16(kp.CipherSuite))
  	if err := b.WriteOpaqueV(kp.InitKey); err != nil {
  		return err
  	}
  	lnb, err := kp.LeafNode.MarshalMLS()
  	if err != nil {
  		return err
  	}
  	b.WriteRaw(lnb) // inline, no length prefix
  	return syntax.WriteVectorV(b, kp.Extensions, func(b *syntax.Builder, e tree.Extension) error {
  		return e.MarshalTo(b)
  	})
  }

  func (kp KeyPackage) marshal(b *syntax.Builder) error {
  	if err := kp.marshalTBS(b); err != nil {
  		return err
  	}
  	return b.WriteOpaqueV(kp.Signature)
  }

  func decodeKeyPackage(c *syntax.Cursor) (KeyPackage, error) {
  	var kp KeyPackage
  	v, err := c.ReadUint16()
  	if err != nil {
  		return kp, err
  	}
  	kp.Version = tree.ProtocolVersion(v)
  	cs, err := c.ReadUint16()
  	if err != nil {
  		return kp, err
  	}
  	kp.CipherSuite = cipher.CipherSuite(cs)
  	if kp.InitKey, err = c.ReadOpaqueV(); err != nil {
  		return kp, err
  	}
  	if kp.LeafNode, err = tree.DecodeLeafNode(c); err != nil {
  		return kp, err
  	}
  	if kp.Extensions, err = syntax.ReadVectorV(c, tree.DecodeExtension); err != nil {
  		return kp, err
  	}
  	if kp.Signature, err = c.ReadOpaqueV(); err != nil {
  		return kp, err
  	}
  	return kp, nil
  }

  // tbsBytes returns the serialized KeyPackageTBS for signing/verification.
  func (kp KeyPackage) tbsBytes() ([]byte, error) {
  	b := syntax.NewBuilder()
  	if err := kp.marshalTBS(b); err != nil {
  		return nil, err
  	}
  	return b.Bytes(), nil
  }

  // VerifySignature checks the KeyPackage signature under label "KeyPackageTBS"
  // using the leaf node's signature key (RFC 9420 §10).
  func (kp KeyPackage) VerifySignature(suite cipher.Suite) (bool, error) {
  	tbs, err := kp.tbsBytes()
  	if err != nil {
  		return false, err
  	}
  	return suite.VerifyWithLabel(kp.LeafNode.SignatureKey, "KeyPackageTBS", tbs, kp.Signature), nil
  }

  // Ref computes the KeyPackageRef = RefHash("MLS 1.0 KeyPackage Reference",
  // KeyPackage) (RFC 9420 §5.2). The value hashed is the full KeyPackage struct
  // serialization (NOT the MLSMessage envelope).
  func (kp KeyPackage) Ref(suite cipher.Suite) ([]byte, error) {
  	raw, err := kp.MarshalMLS()
  	if err != nil {
  		return nil, err
  	}
  	return suite.RefHash("MLS 1.0 KeyPackage Reference", raw)
  }

  // MarshalMLS encodes the KeyPackage to its MLS wire form.
  func (kp KeyPackage) MarshalMLS() ([]byte, error) {
  	b := syntax.NewBuilder()
  	if err := kp.marshal(b); err != nil {
  		return nil, err
  	}
  	return b.Bytes(), nil
  }

  // UnmarshalMLS decodes a KeyPackage, rejecting trailing bytes.
  func (kp *KeyPackage) UnmarshalMLS(data []byte) error {
  	c := syntax.NewCursor(data)
  	v, err := decodeKeyPackage(c)
  	if err != nil {
  		return err
  	}
  	if !c.Empty() {
  		return fmt.Errorf("group: trailing bytes after KeyPackage")
  	}
  	*kp = v
  	return nil
  }
  ```
